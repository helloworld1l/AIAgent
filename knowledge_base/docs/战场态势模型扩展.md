# 战场态势模型扩展

## 1. 目标

本文档用于补齐战场态势域的知识文档、标签体系、术语词典和参数词典，重点支持：

- 红蓝兵力消耗建模
- 态势评估相关建模提示
- 战场感知 / 目标跟踪与态势层关系辨析

本域的核心不是单个平台动力学，而是作战层、编队层、兵力层与感知层的建模。

---

## 2. 领域边界

### 2.1 本域核心问题

战场态势域优先处理：

- 红蓝对抗
- 兵力消耗
- 战损评估
- 态势感知
- 预警、侦察、目标发现与态势更新
- 战场层的目标跟踪与力量变化

### 2.2 与相近领域的边界

容易与战场态势混淆的场景包括：

- “导弹飞行、火箭发射、卫星轨道” → 应优先归到航空航天域
- “鱼雷水下发射、导弹动力学、雷达滤波细节” → 应优先归到军工装备域
- “平台运动学建模”与“战场层态势建模”不是同一层级

如果用户同时提到武器平台和战场对抗，必须先澄清：

- 是要建模单个平台动力学
- 还是要建模战场层的兵力、损耗、预警、态势变化

## 7. 典型建模请求与路由提示

### 7.1 应优先路由到 `lanchester_battle_attrition`

- 构建红蓝双方兵力消耗模型
- 生成战场对抗中的兵力损耗 MATLAB 脚本
- 给定双方初始兵力与杀伤系数，分析作战结果

### 7.2 应优先路由到 `radar_target_tracking_2d`

- 做一个战场预警场景中的目标跟踪模型
- 用雷达观测噪声与过程噪声构建目标跟踪
- 生成二维航迹跟踪估计脚本

### 7.3 应先澄清的表达

- 战场态势模型
- 作战评估模型
- 预警模型
- 目标感知模型
- 对抗模型

需要先澄清：

- 关注的是兵力消耗、目标跟踪，还是更高层的态势推演？
- 是想做单目标跟踪，还是群体对抗演化？
- 是平台级模型，还是战场层模型？

## 8. 反误配规则

### 8.1 不应把兵力消耗误配为武器平台动力学

出现以下词时，应优先考虑战场态势：

- 红蓝
- 战损
- 兵力
- 对抗
- 消耗
- 兰彻斯特

### 8.2 不应把态势感知误配为纯航空航天轨迹

如果用户强调：

- 预警
- 侦察
- 探测
- 态势感知
- 目标发现

则需要先判断是战场层感知，还是单一平台运动学。

### 8.3 复杂复合请求应先分层澄清

例如：

- 战场态势 + 导弹拦截
- 兵力消耗 + 雷达跟踪
- 预警探测 + 平台机动

这类请求通常包含多个层级，必须先问清主建模目标。

## 10. 使用建议

对于战场态势域，请优先判断用户请求处于哪一层：

1. 兵力层：红蓝对抗、兵力消耗、战损演化
2. 感知层：目标发现、雷达观测、跟踪估计
3. 平台层：导弹、鱼雷、火箭、卫星等单平台动力学

只有在主层级清晰的情况下，才应直接生成；否则先进行任务分层澄清。
