# 三大域标签术语参数总览

## 1. 目的

本文档作为 `knowledge_base/docs/` 的跨域总览，用于统一航空航天、军工装备、战场态势三大域的：

- 领域标签
- 术语体系
- 参数别名
- 路由边界
- 反误配提示

该文档适合用于处理跨域请求、模糊请求和多层级请求。

## 4. 高混淆请求与建议路由

### 4.1 “发射模型”

可能对应：

- 火箭发射
- 导弹发射
- 鱼雷水下发射

建议先澄清：

- 对象是什么？
- 介质是空气还是水？
- 关注的是发射段还是完整轨迹？

### 4.2 “轨迹模型”

可能对应：

- 导弹二维飞行轨迹
- 卫星轨道传播
- 雷达估计得到的航迹

建议先澄清：

- 是平台真实运动轨迹，还是传感器估计航迹？
- 是在大气层内飞行，还是轨道空间环境？

### 4.3 “跟踪模型”

可能对应：

- 雷达目标跟踪
- 战场态势感知中的目标跟踪

建议先澄清：

- 是否有雷达 / 传感器观测噪声？
- 是单目标状态估计，还是态势层感知？

### 4.4 “对抗模型”

可能对应：

- 红蓝兵力消耗
- 武器平台拦截对抗

建议先澄清：

- 是战场层对抗，还是单平台动力学拦截？
- 目标是兵力演化，还是轨迹与命中？

| 阻力系数 | 阻力系数, Cd | drag coefficient |
| 发射角 | 仰角, 初始角度 | launch angle |
| 初始速度 | 初速 | init speed, v0 |

## 6. 统一参数归一化建议

当用户使用自然语言描述参数时，可优先归一化到以下规范参数名：

### 6.1 动力学参数

- 质量 → `mass` 或 `mass0`
- 推力 → `thrust`
- 阻力系数 / Cd → `drag_coeff`
- 面积 / 截面积 → `area`
- 时间步长 / 步长 → `dt`
- 仿真时长 / 总时长 → `stop_time`

### 6.2 飞行与轨道参数

- 发射角 / 仰角 → `launch_angle_deg`
- 初始速度 / 初速 → `init_speed` 或 `v0`
- 轨道高度 → `altitude0`
- 引力参数 → `mu`
- 推进时间 → `burn_time`

### 6.3 跟踪估计参数

- 横向速度 / vx → `target_speed_x`
- 纵向速度 / vy → `target_speed_y`
- 观测噪声 → `measurement_noise`
- 过程噪声 → `process_noise`
- 步数 → `steps`

### 6.4 战场对抗参数

- 红方初始兵力 → `red0`
- 蓝方初始兵力 → `blue0`
- 蓝方杀伤系数 → `alpha`
- 红方杀伤系数 → `beta`

## 7. 推荐澄清问题模板

### 7.1 面向对象澄清

- 你说的是火箭、导弹、鱼雷、卫星，还是战场兵力模型？

### 7.2 面向环境澄清

- 场景是在空气中、水下，还是太空轨道环境？

### 7.3 面向层级澄清

- 你要建模的是单个平台动力学，还是战场层的态势与兵力演化？

### 7.4 面向关键参数澄清

- 目前你能提供哪些关键参数，例如质量、推力、发射角、轨道高度、观测噪声、兵力规模？

---

## 8. 知识库扩展建议

后续继续扩展时，优先保持以下一致性：

1. 文档中的标签尽量与 `domain_tags`、`template_family` 对齐
2. 参数名尽量与现有 `default_params`、槽位字段对齐
3. 对每个新场景同时补：
   - 场景说明
   - 标签
   - 术语词典
   - 参数词典
   - 反误配提示
4. 多层级请求一定加“先澄清主目标”的提示

这样可以同时提升：

- BM25 词法检索命中率
- 向量检索的语义覆盖
- 半开放域下的匹配稳定性
- 结构化槽位收集的自然语言兼容性
