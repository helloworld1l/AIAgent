  - 这个项目的知识库是“双份”结构：
      - 本地 JSON 索引：knowledge_base/matlab_knowledge_index.json，由 knowledge_base/builder.py:182 写出
      - Qdrant 向量库：开启 --with-qdrant 时写入，见 knowledge_base/builder.py:188
  - 运行时检索也是混合的：本地文档 + BM25 常驻内存，向量检索优先走 Qdrant，失败再回退本地向量，见 knowledge_base/
    rag_retriever.py:59、knowledge_base/rag_retriever.py:508

  容量跟什么有关

  - 文档条数 / 分块数：Qdrant 里每个文档块就是一个 point，数量越多，占用越大
  - 向量维度：由嵌入模型决定，构建时动态读取，见 knowledge_base/builder.py:54
  - 每条文本长度：会影响 payload 大小、编码时间、BM25 内存占用
  - payload 元数据多少：Qdrant 里除了 vector，还存了 text 和 payload，见 knowledge_base/builder.py:218
  - 索引方式：Qdrant 的向量索引本身也有额外开销
  - 机器资源：磁盘、内存、CPU；本项目 .env 默认嵌入在 cpu 上，见 .env:14

  你可以粗略理解为：

  - Qdrant 存储量 ≈ 点数 × 向量维度 × 4 字节 + payload + 索引开销
  - 本地检索内存量 ≈ 文档总文本量 + BM25 词项索引

  知识扩容流程

  - 1. 增加知识源
      - 现在最直接是改 knowledge_base/matlab_model_data.py
      - 给模型补更多 keywords、description、examples、default_params
  - 2. 扩展文档生成逻辑
      - 当前 prepare_documents() 只生成 model 和 example 两类，见 knowledge_base/builder.py:157、knowledge_base/
        builder.py:174
      - 如果你要扩容成“理论说明 / 参数指南 / 常见错误 / 修复案例”，就需要改这里
  - 3. 重建本地索引
      - 执行 python main.py build，入口见 main.py:13
      - 会生成本地 matlab_knowledge_index.json
  - 4. 重建 Qdrant 向量库
      - 执行 python main.py build --with-qdrant
      - 这一步会编码文本、建 collection、写入 points，见 knowledge_base/builder.py:203、knowledge_base/builder.py:225
  - 5. 启动服务
      - 执行 python main.py api，入口见 main.py:28

  是不是增量扩容

  - 当前实现不是“增量 upsert 优先”，而是整库重建
  - 因为 build_qdrant_index() 里先 delete_collection，再 create_collection，见 knowledge_base/builder.py:197、
    knowledge_base/builder.py:203
  - 所以你每次扩容后，推荐直接重新 build 一次

  运行时到底怎么用这些数据

  - 先加载本地 JSON，并和内置 catalog 合并，见 knowledge_base/rag_retriever.py:62、knowledge_base/rag_retriever.py:67
  - 本地始终会建 BM25 索引，见 knowledge_base/rag_retriever.py:42
  - 向量后端优先 Qdrant，成功则 backend=qdrant，见 knowledge_base/rag_retriever.py:264
  - Qdrant 不可用时，回退本地向量，见 knowledge_base/rag_retriever.py:282、knowledge_base/rag_retriever.py:399

  一句话回答你的问题

  - 知识扩容流程：补数据 → 生成文档 → 本地 JSON 重建 → 可选写入 Qdrant → 重启 API
  - 存储位置：会存到 Qdrant，但同时也存本地 JSON；Qdrant 不是唯一存储
