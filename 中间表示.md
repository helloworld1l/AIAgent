› IR是什么


• IR 是 Intermediate Representation，中文一般叫 中间表示。

  简单说：

  - 它是用户自然语言和最终 MATLAB 代码之间的“结构化桥梁”
  - 作用是把“我要一个潜艇水下发射鱼雷模型，考虑阻力”翻译成机器更容易处理的标准结构
  - 然后代码生成器再根据这个结构去产出 .m 文件

  为什么需要 IR

  - 直接从自然语言生成代码，容易跳步骤、漏假设、乱选模板
  - 有了 IR，可以先把“建模意图”表达清楚，再生成代码
  - IR 也方便做校验、修复、补参数、追踪错误来源

  在你这个项目里，如果引入 IR，可能长这样

  - 对象：submarine、torpedo
  - 场景：underwater_launch
  - 动力学：1d_translation 或 6dof
  - 力项：thrust、buoyancy、hydrodynamic_drag、gravity
  - 状态量：position、velocity
  - 参数：mass、water_density、drag_coeff、area
  - 事件：launch_start、motor_ignite
  - 输出：trajectory、velocity_curve

  比如一个很简化的 IR 可以是：

  {
    "domain": "underwater_weapon",
    "scenario": "torpedo_underwater_launch",
    "entities": ["torpedo"],
    "dynamics": {
      "type": "1d_translation",
      "states": ["x", "v"]
    },
    "forces": ["thrust", "hydrodynamic_drag", "buoyancy", "gravity"],
    "parameters": {
      "mass": 120,
      "drag_coeff": 0.4,
      "water_density": 1000,
      "area": 0.03
    },
    "outputs": ["x", "v"],
    "simulation": {
      "stop_time": 20,
      "dt": 0.01
    }
  }

  然后生成器再把它变成 MATLAB 代码。

  你可以把 IR 理解成

  - 不是最终代码
  - 也不是原始自然语言
  - 而是“已经理解过一次的结构化模型说明书”

  它和你现在的 ModelSpec 的关系

  - 你现在项目里的 ModelSpec 已经有一点 IR 的味道
  - 但它还是偏“模板参数单”，核心还是依赖 model_id
  - 真正开放域 IR 会更强调：
      - 系统由哪些对象组成
      - 用什么动力学方程
      - 有哪些物理作用
      - 状态/输入/输出是什么
      - 哪些是假设，哪些参数缺失

  一句话

  - 现在的 model_id 更像“选菜单”
  - IR 更像“写菜谱”
